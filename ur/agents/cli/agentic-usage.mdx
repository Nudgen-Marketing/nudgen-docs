## جائزہ

Nudgen CLI کو **AI-First** اپروچ کے ساتھ ڈیزائن کیا گیا ہے۔ اگرچہ یہ انسانوں کے لیے ایک خوبصورت TUI فراہم کرتی ہے، لیکن ہر کمانڈ کو مشین سے مشین مواصلت کے لیے بھی بہتر بنایا گیا ہے۔

### ایجنٹک پیٹرن (The Agentic Pattern)

جب کوئی AI ایجینٹ (جیسے Claude، GPT، یا کوئی مخصوص آٹومیشن ایجینٹ) CLI استعمال کرتا ہے، تو اسے ان پیٹرنز کی پیروی کرنی چاہیے:

1. **JSON آؤٹ پٹ**: مشین کے ذریعے پارس کرنے کے قابل ڈیٹا حاصل کرنے کے لیے ہر کمانڈ کے ساتھ `--json` شامل کریں۔
2. **سیاق و سباق سے آگاہی**: آپریشنز کرنے سے پہلے ہمیشہ ٹیم کے سیاق و سباق کی تصدیق کریں۔
3. **غیر انٹرایکٹو**: انٹرایکٹو TUI پرامپٹس سے بچنے کے لیے فلیگز (flags) استعمال کریں۔

## JSON آٹومیشن

ہر فہرست (list) اور اسٹیٹس کمانڈ `--json` فلیگ کی حمایت کرتی ہے۔ یہ فارمیٹ شدہ ٹیبل کے بجائے ایک صاف JSON ڈھانچہ واپس کرتی ہے۔

```bash
# تمام مہمات کو بطور JSON حاصل کریں
nudgen campaigns list --json

# موجودہ ٹیم میں رابطے حاصل کریں
nudgen contacts list --json
```

## ایجینٹس کے لیے ملٹی ٹیننٹ (Multi-tenant) ورک فلو

چونکہ Nudgen متعدد ٹیموں کی حمایت کرتا ہے، اس لیے ایجینٹ کو وسائل بنانے یا اپ ڈیٹ کرنے سے پہلے یہ یقینی بنانا چاہیے کہ وہ صحیح ٹیم پر کام کر رہا ہے۔

**تجویز کردہ ورک فلو:**

1. **موجودہ ٹیم چیک کریں**:
   ```bash
   nudgen teams current --json
   ```
2. **ٹیم سوئچ کریں (اگر ضرورت ہو)**:
   اگر پچھلے مرحلے سے حاصل کردہ `id` ہدف ٹیم سے میل نہیں کھاتی ہے، تو سوئچ کریں:
   ```bash
   nudgen teams switch <target-team-id>
   ```
3. **ڈیٹا کمانڈ چلائیں**:
   `campaigns` ، `contacts` ، یا `brand` کمانڈز کے ساتھ آگے بڑھیں۔

## ایجینٹ ڈویلپرز کے لیے بہترین طریقے

- **ایرر ہینڈلنگ**: CLI کمانڈز کے ایگزٹ کوڈ (exit code) کی نگرانی کریں۔ غیر صفر ایگزٹ کوڈ غلطی (تصدیق، نیٹ ورک، یا توثیق) کی نشاندہی کرتا ہے۔
- **ریٹ لمٹنگ (Rate Limiting)**: CLI Nudgen API کے ساتھ بات چیت کرتی ہے۔ یقینی بنائیں کہ آپ کے ایجینٹ اسکرپٹس ممکنہ ریٹ لمٹس کو بہتر طریقے سے سنبھالتے ہیں۔
- **خاموش موڈ**: اگر کوئی کمانڈ ہر پرامپٹ کے لیے مخصوص فلیگ کی حمایت نہیں کرتی ہے، تو یقینی بنائیں کہ آپ کا ایجینٹ TUI لوپ میں پھنسے بغیر stdout/stderr اسٹریمز کو سنبھال سکتا ہے۔
