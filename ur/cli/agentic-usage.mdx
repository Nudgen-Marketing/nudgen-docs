## جائزہ

Nudgen CLI کو **AI-First** سوچ کے ساتھ ڈیزائن کیا گیا ہے۔ جہاں یہ انسانوں کے لیے ایک خوبصورت TUI فراہم کرتا ہے، وہیں ہر کمانڈ مشین ٹو مشین رابطے کے لیے بھی بہترین ہے۔

### ایجنٹک پیٹرن (Agentic Pattern)

جب کوئی AI ایجنٹ (جیسے Claude، GPT، یا کوئی خاص آٹومیشن ایجنٹ) اس CLI کا استعمال کرتا ہے، تو اسے ان پیٹرنز پر عمل کرنا چاہیے:

1. **JSON آؤٹ پٹ**: مشینی طور پر پڑھے جانے والے ڈیٹا کے لیے ہر کمانڈ کے ساتھ `--json` لگائیں۔
2. **تناظر کی آگاہی**: کام شروع کرنے سے پہلے ہمیشہ ٹیم کے تناظر کی تصدیق کریں۔
3. **غیر انٹرایکٹو**: انٹرایکٹو یوزر انٹرفیس کے سوالات سے بچنے کے لیے مناسب فلیگز (Flags) استعمال کریں۔

## JSON آٹومیشن

ہر فہرست (List) اور صورتحال (Status) والی کمانڈ `--json` فلیگ کو سپورٹ کرتی ہے۔ اس سے فارمیٹ شدہ ٹیبل کے بجائے ایک صاف ستھرا JSON اسٹرکچر ملتا ہے۔

```bash
# تمام مہمات کی فہرست JSON میں حاصل کریں
nudgen campaigns list --json

# موجودہ ٹیم کے رابطے حاصل کریں
nudgen contacts list --json
```

## ایجنٹوں کے لیے ملٹی ٹیم ورک فلو

چونکہ Nudgen متعدد ٹیموں کو سپورٹ کرتا ہے، ایجنٹ کو وسائل بنانے یا اپ ڈیٹ کرنے سے پہلے یہ یقینی بنانا چاہیے کہ وہ درست ٹیم پر کام کر رہا ہے۔

**تجویز کردہ ورک فلو:**

1. **موجودہ ٹیم چیک کریں**:
   ```bash
   nudgen teams current --json
   ```
2. **ٹیم تبدیل کریں (اگر ضرورت ہو)**:
   اگر پچھلے مرحلے سے ملنے والی `id` ہدف ٹیم سے مختلف ہے، تو سوئچ کریں:
   ```bash
   nudgen teams switch <target-team-id>
   ```
3. **ڈیٹا کمانڈ چلائیں**:
   کمانڈز جیسے `campaigns` یا `contacts` کے ساتھ کام جاری رکھیں۔

## ایجنٹ ڈویلپرز کے لیے بہترین مشورے

- **غلطیوں کا انتظام (Error Handling)**: ایگزٹ کوڈ (Exit Code) کی نگرانی کریں۔ غیر صفر کوڈ غلطی (نیٹ ورک، یا لاگ ان وغیرہ) کی نشاندہی کرتا ہے۔
- **درخواست کی حد (Rate Limiting)**: یہ CLI براہ راست Nudgen API سے بات کرتا ہے۔ یقینی بنائیں کہ آپ کے ایجنٹ اسکرپٹس درخواستوں کی حد کو سمجھتے ہیں۔
- **سائلنٹ موڈ**: اگر کسی کمانڈ میں ہر سوال کے لیے مخصوص فلیگ موجود نہیں ہے، تو یقینی بنائیں کہ آپ کا ایجنٹ TUI کے چکر میں پھنسے بغیر کام مکمل کر سکے۔
