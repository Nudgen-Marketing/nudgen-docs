## Autentikasi

Semua permintaan API memerlukan Personal Access Token (PAT) yang diteruskan melalui header `Authorization`. Anda dapat membuat token dari dasbor Nudgen di bawah **Pengaturan → API Keys**.

```bash
curl -X GET https://app.nudgen.com/api/v1/user/me \
  -H "Authorization: Bearer <your-pat>"
```

## URL Dasar

URL dasar untuk semua permintaan API adalah:
`https://app.nudgen.com`

## Titik Akhir (Endpoints)

### Autentikasi & Identitas
- `GET /api/v1/user/me`: Dapatkan pengguna terautentikasi saat ini.
- `GET /api/v1/teams`: Daftar semua tim untuk pengguna saat ini.
- `POST /api/v1/teams/switch`: Ganti konteks tim yang aktif.

### Kontak
- `GET /api/v1/contacts`: Daftar kontak dengan paginasi, pencarian, dan filter tag.
- `POST /api/v1/contacts/add`: Buat kontak baru.
- `GET /api/v1/contacts/:id`: Ambil kontak tertentu.
- `PATCH /api/v1/contacts/:id`: Perbarui kontak yang ada.
- `DELETE /api/v1/contacts/:id`: Hapus kontak.

### Kampanye
- `GET /api/v1/campaigns`: Daftar kampanye di tim saat ini.
- `POST /api/v1/campaigns`: Buat kampanye baru.
- `GET /api/v1/campaigns/:id`: Dapatkan detail kampanye.
- `GET /api/v1/campaigns/:id/stats`: Dapatkan statistik pengiriman kampanye.

### Templat
- `GET /api/v1/templates`: Daftar templat email.
- `POST /api/v1/templates`: Buat templat email baru.

### Pengaturan
- `GET /api/v1/settings/brand`: Dapatkan konfigurasi merek.
- `PATCH /api/v1/settings/brand`: Perbarui pengaturan merek.
