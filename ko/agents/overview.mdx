## 소개

Nudgen은 에이전트 시대(Agentic Era)를 위해 구축되었습니다. **Claude Code**, **Codex**, **Cursor**, **OpenClaw**와 같은 AI 에이전트가 마케팅 스택과 상호 작용할 수 있는 구조화된 방법을 제공합니다.

새 앱에 Nudgen을 설치하거나, 기존 공급업체에서 마이그레이션하거나, 복잡한 캠페인 워크플로우를 자동화해야 하는 경우, 에이전트 우선 인터페이스를 통해 높은 신뢰성과 개발 효율성을 보장합니다.

## 사용 사례

에이전트 기반 마케팅은 실제로 어떻게 작동할까요? 몇 가지 일반적인 워크플로우를 소개합니다:

### "터미널에서 이 트랜잭션 이메일 테스트해줘"
**에이전트가 수행하는 작업:** CLI를 사용하여 템플릿 ID 및 샘플 데이터 변수가 포함된 테스트용 트랜잭션 이메일을 보냅니다. 응답 상태 및 전송 확인을 표시합니다.

**사용자가 완료하는 작업:** 받은 편지함을 확인하고 렌더링된 이메일을 검토한 후 준비가 되면 템플릿을 게시합니다.

---

### "내 이메일 설정을 감사하고 Nudgen에 매핑해줘"
**에이전트가 수행하는 작업:** 코드베이스를 스캔하여 이메일 공급업체 호출을 찾고 모든 전송, 템플릿 및 웹훅을 식별합니다. 각 항목을 Nudgen의 해당 기능에 매핑하고 파일별 마이그레이션 계획을 생성합니다.

**사용자가 완료하는 작업:** Nudgen 에디터에서 템플릿을 다시 생성하고 대시보드에서 수신 거부 및 웹훅 설정을 구성합니다.

---

### "Nudgen에 취소 이벤트 전송해줘"
**에이전트가 수행하는 작업:** 구독이 취소될 때 Nudgen 이벤트 API를 호출하도록 Stripe 웹훅 핸들러를 연결합니다. 이벤트 이름을 설정하고 관련 속성(플랜, 이유, 날짜)을 매핑합니다.

**사용자가 완료하는 작업:** 대시보드에서 취소 Nudgen(플로우)을 생성하고 필요한 경우 사용자 정의 연락처 속성을 정의합니다.

---

### "내 Next.js 앱에 Nudgen 설치해줘"
**에이전트가 수행하는 작업:** Next.js를 감지하고, JavaScript SDK를 설치하고, .env에서 NUDGEN_API_KEY를 연결하고, 연락처 동기화 루트 및 트랜잭션 전송 예제를 생성하고, 테스트 체크리스트를 출력합니다.

**사용자가 완료하는 작업:** Nudgen 대시보드에 API 키를 추가하고, 트랜잭션 이메일 템플릿을 게시하고, 테스트 전송을 확인합니다.
