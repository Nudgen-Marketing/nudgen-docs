## 이 페이지의 도움말
연락처가 어떻게 저장되는지, 왜 마스킹되어 표시되는지, 세그먼트화를 위해 태그를 어떻게 사용하는지 설명합니다. 이를 통해 안심하고 연락처를 업로드하고 캠페인을 빌드할 수 있습니다.

## 시작하기 전에
- Nudgen 계정
- 최소 하나의 이메일 주소가 포함된 CSV 파일 (형식은 [업로드 가이드](/ko/contacts/upload) 참조)

## 연락처 데이터 저장 방식
Nudgen은 저장되는 모든 개인 식별 정보(PII)를 저장 전 암호화합니다:

- **이메일** — AES-256 암호화. 저희도 읽을 수 없습니다.
- **이름** — 제공된 경우 암호화됩니다. CSV에서 선택 사항입니다.
- **사용자 정의 메타데이터** — 업로드하는 모든 추가 컬럼도 함께 암호화됩니다.

또한 중복 방지를 위해 각 이메일의 단방향 해시를 저장합니다. 동일한 리스트 내의 중복 이메일은 자동으로 통합됩니다.

## 마스킹 뷰 (Masked View)
기본적으로 연락처 리스트는 실제 주소 대신 `j***@g***.com`과 같은 마스킹된 값을 보여줍니다. 개인 식별 정보는 저장 시 암호화되며, 저희는 평문 이메일이나 이름을 보거나 저장하지 않습니다. 마스킹은 주변 사람들에게 데이터가 노출되는 것을 방지하고, 화면 공유 시의 노출 위험을 줄여줍니다.

**연락처 확인:** 귀하(및 접근 권한이 있는 워크스페이스 멤버)는 **확인(Reveal)** 버튼을 클릭하여 UI에서 실제 이메일과 이름을 볼 수 있습니다. 복호화는 확인 버튼을 누를 때 브라우저 내에서만 발생하며, 페이지를 이동하거나 새로고침하면 다시 마스킹됩니다. 분석, 로그 또는 내보내기 파일에서는 복호화된 개인 정보를 절대 표시하지 않습니다.

## 태그 및 세그먼트화
연락처를 업로드하거나 나중에 수정할 때 태그를 지정하세요. 캠페인에서 특정 세그먼트를 타겟팅할 때 태그를 사용합니다:

- `inactive` — 30일 이상 로그인하지 않은 사용자
- `vip` — 가치가 높거나 프리미엄 서비스를 이용하는 고객
- `trial` — 무료 체험 중인 사용자

캠페인을 생성할 때 전체 연락처 또는 특정 태그가 있는 연락처 중에서 대상을 선택할 수 있습니다. 태그를 사용하면 세그먼트별로 서로 다른 시퀀스를 쉽게 운영할 수 있습니다.

## 연락처 상태

연락처는 다음 중 하나의 상태를 가집니다:

- **Active (활성)** — 캠페인을 수신할 수 있는 상태입니다. 새로 가져온 연락처의 기본 설정입니다.
- **Unsubscribed (수신 거부)** — 수신 거부 링크를 통하거나 수동으로 거부한 상태입니다. 상태를 변경하지 않는 한 이후의 캠페인 이메일을 받지 않습니다.
- **Suppressed / Bounced (도달 불가)** — 전송 실패(잘못된 주소, 편지함 가득 참 또는 차단 리스트 포함)된 상태입니다. 도달률 보호를 위해 향후 발송 대상에서 자동으로 제외됩니다. 저희 시스템은 발송 대행사의 차단 리스트와 실시간으로 동기화됩니다.
