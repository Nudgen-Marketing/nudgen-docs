## 本页可以帮助你做什么
了解某场活动的实际表现，包括发送量、送达量、打开量、点击量，以及每个收件人的互动情况。你也可以从中判断活动是否存在送达问题。

## 开始前
- 工作区中至少已经有一个已发送或正在发送的活动
- 你可以打开活动详情页查看分析

## 指标卡片
活动详情页通常会显示以下指标：

| 指标 | 含义 |
| --- | --- |
| **发送尝试数** | 所有发送尝试的总数，包括失败请求。 |
| **成功送达数** | 被收件方邮件服务器成功接受的邮件数量。 |
| **打开次数** | 邮件被打开的次数，通过追踪像素统计。 |
| **点击次数** | 邮件中的链接被点击的次数。 |
| **点击率** | `(点击次数 / 发送尝试数) × 100`。 |
| **垃圾邮件投诉** | 收件人将邮件标记为垃圾邮件的次数。 |
| **软退信** | 临时投递异常，例如邮箱已满或服务暂时不可用。 |
| **硬退信** | 永久投递失败，例如地址无效或域名无法接收。 |

## 送达成功率与失败率
- **成功率** = `(成功送达数 / 发送尝试数) × 100`
- **失败率** = `((退信数 + 失败数) / 发送尝试数) × 100`

如果失败率偏高，建议检查联系人质量，并清理无效地址，以免影响后续发件信誉。

## 趋势图
图表会按天、周或月展示活动表现。你可以切换时间粒度，并选择只看打开、点击或送达等某一项指标。

## 日期范围
你可以通过日期范围筛选数据。默认通常覆盖整个活动周期，也可以改成只看最近一周或某个特定时间段。

## 收件人活动表
你还可以查看每个联系人的状态和行为，例如：
- **联系人**：邮箱和姓名，默认脱敏显示
- **状态**：已发送、已送达、已打开、已点击、已退信等
- **发送时间**
- **送达时间**
- **首次打开时间**
- **首次点击时间**

这些信息常用于：
- 找出高互动用户，安排后续跟进
- 识别需要清理的退信地址
- 排查某个具体联系人为什么没有收到邮件

## 接下来可以做什么
- 根据分析结果优化后续活动的主题行、语气和发送时间
- 删除持续退信的联系人，保护送达率
- 如果已接近套餐上限，可以去[账单概览](/cn/billing/overview)查看发送限制
